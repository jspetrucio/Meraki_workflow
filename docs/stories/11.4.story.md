## Story

**As a** network architect,
**I want** to manage configuration templates and bind/unbind networks,
**so that** I can standardize configurations across multiple sites.

---

## Acceptance Criteria

1. `discover_templates` returns all templates with bound network counts
2. `bind_network` associates a network with a template (with auto-bind options)
3. `unbind_network` disassociates a network (preserving or removing config)
4. Template CRUD fully supported (create, update, delete)
5. Backup created before any template binding/unbinding operation
6. Dry-run mode supported: preview template binding changes before applying
7. Template data integrated into `full_discovery()` output

## Implementation Pattern

This story follows the standardized 7-step implementation pattern:

1. `api.py` -- Add SDK wrapper methods with `@log_api_call`
2. `discovery.py` -- Add discovery function with `client.safe_call()`
3. `config.py` -- Add config function with backup + `ConfigResult`
4. `agent_router.py` -- Register in `FUNCTION_REGISTRY`
5. `agent_tools.py` -- Add tool schemas + TOOL_SAFETY classification
6. `safety.py` -- Register in `SAFETY_CLASSIFICATION`
7. `tests/` -- Add comprehensive tests

**Note:** Both `SAFETY_CLASSIFICATION` (safety.py) and `TOOL_SAFETY` (agent_tools.py) must be updated.
