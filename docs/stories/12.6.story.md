## Story

**As a** wireless engineer,
**I want** to manage per-SSID firewall rules (L3/L7),
**so that** I can apply traffic filtering at the wireless level through natural language.

---

## Acceptance Criteria

1. `discover_ssid_firewall` returns L3 and L7 rules per SSID
2. `configure_ssid_firewall` supports both L3 (IP/port) and L7 (application) rules
3. Backup + rollback per SSID
4. Backup created before any SSID firewall change
5. Dry-run mode supported: preview rule changes before applying
6. SSID firewall status integrated into `full_discovery()` output
7. Safety: MODERATE classification (affects wireless traffic)
