## Story

**As a** network admin,
**I want** to manage device inventory (claim/release),
**so that** I can onboard and offboard devices via CNL.

---

## Acceptance Criteria

1. `discover_inventory` returns full org inventory with claim status
2. `claim_device` claims a device by serial/order number
3. `release_device` releases a device from the org (requires confirmation)
4. Natural language: "claim device Q2XX-XXXX-XXXX" works

## Implementation Pattern

This story follows a simplified implementation pattern (no config layer):

1. `api.py` -- Add SDK wrapper methods with `@log_api_call`
2. `discovery.py` -- Add discovery function with `client.safe_call()`
3. `agent_router.py` -- Register in `FUNCTION_REGISTRY`
4. `agent_tools.py` -- Add tool schemas + TOOL_SAFETY classification
5. `safety.py` -- Register in `SAFETY_CLASSIFICATION`
6. `tests/` -- Add comprehensive tests

**Note:** Both `SAFETY_CLASSIFICATION` (safety.py) and `TOOL_SAFETY` (agent_tools.py) must be updated.
