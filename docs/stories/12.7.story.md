## Story

**As a** network admin,
**I want** to configure captive portal splash pages for SSIDs,
**so that** I can customize the guest WiFi experience through natural language.

---

## Acceptance Criteria

1. `discover_splash_config` returns splash page type and settings per SSID
2. `configure_splash_page` supports click-through, sign-on, and sponsored guest modes
3. Backup created before any splash page change
4. Dry-run mode supported: preview splash changes before applying
5. Splash page status integrated into `full_discovery()` output
6. Safety: MODERATE classification (affects guest access)

3. `config.py` -- Add config function with backup + `ConfigResult`
